Temporary interface into to db.

When entering your request params, you would need to encode them the way they are supposed to be transmitted. E.g. for GET, you would write "/test-url?test=1&hello=world".


Body for POST/PUT requests should be in json format e.g. {a: 1}

Only POST works at the moment. You have 2 options: 1. Supply the table schema. 2. Ommit the table schema and primary key will be generated for you with name "_id". Example of the request payload without schema {"data":{"_id": "1","my": "test"}}. And with schema {"data":{"_id": "1","my": "test"}, "schema": "++a"}.
